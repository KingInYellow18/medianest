    Quick Diagnostics¶

    System Health Check¶

    Quick Status Commands¶

    # Check all services status
curl http://localhost:4000/api/v1/health

# Check specific service components
curl http://localhost:4000/api/v1/health/database
curl http://localhost:4000/api/v1/health/redis
curl http://localhost:4000/api/v1/health/external

# Check frontend accessibility
curl http://localhost:3000

# Check process status
ps aux | grep -E "(node|npm|next)"

    Service Verification Checklist¶

    # 1. Check ports are listening
netstat -tlnp | grep -E "(3000|4000|5432|6379)"

# 2. Check environment variables
env | grep -E "(DATABASE_URL|REDIS_URL|JWT_SECRET)"

# 3. Check log files for errors
tail -f backend/logs/app.log
tail -f frontend/.next/build.log

# 4. Check disk space
df -h

# 5. Check memory usage
free -m

    Installation Issues¶

    Node.js and npm Issues¶

    Problem: Node Version Mismatch¶

    # Symptoms
Error: Node.js version 14.x is not supported
npm ERR! engine Unsupported engine

# Solution
# Install Node.js 18+
nvm install 18
nvm use 18
node --version  # Should show 18.x.x

    Problem: npm Install Failures¶

    # Symptoms
npm ERR! code EACCES
npm ERR! peer dep missing

# Solutions
# Clear npm cache
npm cache clean --force

# Delete node_modules and reinstall
rm -rf node_modules package-lock.json
npm install

# Fix permissions (Linux/Mac)
sudo chown -R $(whoami) ~/.npm

    Database Setup Issues¶

    Problem: PostgreSQL Connection Failed¶

    # Symptoms
Error: connect ECONNREFUSED 127.0.0.1:5432
Database connection failed

# Diagnostics
# Check if PostgreSQL is running
sudo systemctl status postgresql
# or on Mac
brew services list | grep postgresql

# Check connection
psql -h localhost -U postgres -c "SELECT 1;"

# Solutions
# Start PostgreSQL
sudo systemctl start postgresql
# or on Mac
brew services start postgresql

# Check DATABASE_URL format
echo $DATABASE_URL
# Should be: postgresql://user:password@host:port/database

    Problem: Prisma Migration Issues¶

    # Symptoms
Migration failed: column does not exist
Schema drift detected

# Solutions
# Reset database (development only)
npx prisma migrate reset

# Generate Prisma client
npx prisma generate

# Deploy pending migrations
npx prisma migrate deploy

# Check migration status
npx prisma migrate status

    Error Code Reference¶

    HTTP Error Codes¶

    4xx Client Errors¶

    400 Bad Request:
  - Invalid JSON payload
  - Missing required fields
  - Validation errors

401 Unauthorized:
  - Missing authentication token
  - Invalid JWT token
  - Expired session

403 Forbidden:
  - Insufficient permissions
  - Access denied
  - Rate limit exceeded

404 Not Found:
  - Endpoint not found
  - Resource not found
  - User not found

409 Conflict:
  - Duplicate entry
  - Resource conflict
  - Version mismatch

429 Too Many Requests:
  - Rate limit exceeded
  - API quota exceeded
  - Too many login attempts

    5xx Server Errors¶

    500 Internal Server Error:
  - Unhandled exception
  - Database error
  - External service error

502 Bad Gateway:
  - Upstream server error
  - Proxy configuration error
  - Service unavailable

503 Service Unavailable:
  - Database connection failed
  - External service down
  - Maintenance mode

504 Gateway Timeout:
  - Request timeout
  - Database timeout
  - External API timeout

    Log Analysis¶

    Log Location Guide¶

    Backend Logs¶

    # Application logs
tail -f backend/logs/app.log
tail -f backend/logs/error.log

# PM2 logs (production)
pm2 logs medianest

# Docker logs
docker logs medianest-backend

    System Logs¶

    # Nginx logs
tail -f /var/log/nginx/access.log
tail -f /var/log/nginx/error.log

# PostgreSQL logs
tail -f /var/log/postgresql/postgresql-14-main.log

# Redis logs
tail -f /var/log/redis/redis-server.log

    Log Analysis Commands¶

    Common Log Patterns¶

    # Find authentication errors
grep -i "authentication" logs/app.log | tail -20

# Find database errors
grep -i "database\|prisma" logs/app.log | tail -20

# Find API errors
grep -E "40[0-9]|50[0-9]" logs/app.log | tail -20

# Find slow queries
grep -i "slow query" logs/app.log

# Find memory issues
grep -i "memory\|heap" logs/app.log

# Get error summary
grep -i "error" logs/app.log | cut -d' ' -f4- | sort | uniq -c | sort -nr

    Log Analysis Tools¶

    Structured Log Analysis¶

    # Using jq for JSON logs
cat logs/app.log | jq 'select(.level == "error")'

# Filter by timestamp
cat logs/app.log | jq 'select(.timestamp > "2025-09-07T10:00:00")'

# Group by error type
cat logs/app.log | jq -r '.message' | sort | uniq -c | sort -nr
